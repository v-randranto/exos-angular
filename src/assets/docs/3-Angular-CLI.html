Générer un projet Angular avec le CLI

Documentation

Angular CLI

Créer un nouveau projet

Pour créer un nouveau projet on exécute la commande ng new <NOM_DU_DOSSIER>. Cette commande :

  • crée le dossier spécifié,
  • installe localement les dépendances (paquets npm),
  • crée un squelette d'application Angular prête à fonctionner.
ng new app-demo

Documentation de la commande new.

Le programme vous posera 2 questions :

"Would you like to add Angular routing" -> Yes
"Which stylesheet format would you like to use" -> SCSS

Le CLI installe ensuite les dépendances, ce qui peut prendre plusieurs minutes.

Pour travailler dans le dossier du projet on se place dans le dossier créé, avec la commande cd :

cd app-demo

On peut obtenir l'aide du programme avec la commande :

ng help

On peut aussi obtenir de l'aide sur une fonction spécifique en indiquant le nom de la commande (ci-dessous generate) suivi de l'option --help :

ng generate --help

Démarrer un serveur de test

Pour tester l'application en cours de développement, le CLI peut démarrer un petit server HTTP avec un système de live-reload qui permet de rafraîchir le navigateur à chaque enregistrement d'un fichier source.

ng serve

Par défaut le serveur est disponible à l'adresse http://localhost:4200/.

Pour arrêter le serveur : Ctrl + C dans le terminal qui exécute le serveur.

Avant de commencer

Lorsqu'on développe une application web on doit toujours ouvrir les outils de développement dans le navigateur et en particulier la console, pour repérer immédiatement :

  • les erreurs de syntaxe (qui font planter l'application)
  • certaines erreurs de logique (valeurs "undefined")
  • les messages informatifs affichés par le navigateur ou le framework.

De même, on garde un œil sur la console du serveur de test, qui peut afficher des informations liées à la compilation TypeScript ou des messages du serveur de test.

Lorsqu'on développe, on teste à chaque modification du code afin de repérer les erreurs le plus tôt possible. Si on ne respecte pas cette règle, on peut découvrir une erreur en aval, qui peut provenir d'une dizaine de modifications faites en amont.

Si on respecte ce principe, (test early, test often) toute erreur provient presque certainement du dernier changement effectué.

Utilisation du CLI d'Angular pour générer des parties de l'application

Le CLI d'Angular permet de créer des applications basées sur des modèles prédfinis mais aussi, à l'intérieur de l'application, de créer des composants et autres types d'éléments d'une application Angular.

Générer un contenu (code scaffolding)

La commande ng generate crée un type de contenu Angular. Le nom du type de structure sera suffixé au nom indiqué. Par exemple la commande:

ng generate service user

Créera un service dont la classe aura pour nom "UserService".

Note : les noms de classes commencent toujours par une majuscule par convention (PascalCase). Les noms de fichiers et de dossiers sont générés avec des tirets (kebab-case).

ng generate component fiche-produit

Documentation de la commande generate.

Générer l'application à publier

Pour générer une version prête à publier (mettre en ligne) on utilisera la commande ng build. Cette commande génère l'application dans un dossier nommé "dist", à la racine du projet. Pour créer une version optimisée prête à mettre en ligne on utilisera le flag --prod.

ng build --prod

Le contenu du dossier ./dist peut être directement mis en ligne sur un hébergement adapté.
